## 技术博客：React 自定义 Hook 的 WebSocket 实时推送设计

### 问题

股票行情需要实时更新，传统轮询方案延迟高、资源浪费。需要一个可靠的 WebSocket 连接管理方案。

### 方案设计

```javascript
// hooks/useWebSocket.js 核心逻辑
function useWebSocket(url) {
  const [data, setData] = useState(null);
  const [status, setStatus] = useState('disconnected');
  const wsRef = useRef(null);

  useEffect(() => {
    const ws = new WebSocket(url);
    wsRef.current = ws;

    ws.onopen = () => setStatus('connected');
    ws.onmessage = (e) => setData(JSON.parse(e.data));
    ws.onclose = () => {
      setStatus('disconnected');
      // 自动重连
      setTimeout(() => reconnect(), 3000);
    };

    return () => ws.close();
  }, [url]);

  return { data, status };
}
```

### 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 重连策略 | 3 秒固定间隔 | 行情场景需要快速恢复，不宜指数退避 |
| 数据格式 | JSON | 前端直接解析，无需额外协议层 |
| 状态管理 | useState + useRef | Hook 内部自管理，不依赖外部 store |
| 断线处理 | 自动重连 + 状态提示 | 用户无感知恢复 |

## 复盘

**对的事：**
- 先搭骨架再填数据——页面结构和路由先跑通，数据后接入
- ThemeContext 统一管理暗色模式——避免每个组件单独处理
- 调研东方财富 API 在写代码之前——确认数据源可行再动手

**不对的事：**
- 前端页面一次性搭了 7 个，但都没有真实数据——应该先做 1-2 个核心页面（首页 + 股票详情）做到可用，再扩展

**今天最大的教训：** 前端骨架搭太多但都是空壳，不如集中精力把 1-2 个页面做精做透。"全而空"不如"少而实"。

## 今日收获

- 东方财富 push2 API 无需认证，可直接调用北向资金/龙虎榜/板块数据
- WebSocket 自动重连用固定间隔（3s）比指数退避更适合行情场景
- React 自定义 Hook 封装 WebSocket 比在组件内直接管理更清晰
- Tailwind 自定义色板要符合 A 股惯例（红涨绿跌）
- 先做 1-2 个核心页面做到可用，再扩展其余页面
