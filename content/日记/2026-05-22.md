## 技术博客：当 SSH 被阻断时

### 问题

公司网络对 SSH 不友好，经常连不上香港 VPS。但 HTTP/HTTPS 正常。

### 方案对比

| 方案 | 可行性 | 风险 |
|------|--------|------|
| 修复 SSH 通道 | 公司网络限制，不可控 | - |
| VPN | 需要额外服务端 | 复杂度增加 |
| HTTP 代理部署 | 完全可行 | 需要加 token 验证防滥用 |

### deploy.php 实现

```php
<?php
// HTTP 文件上传代理 — 绕过 SSH 部署
$token = getenv('DEPLOY_TOKEN');
if ($_SERVER['HTTP_X_DEPLOY_TOKEN'] !== $token) {
    http_response_code(403);
    exit;
}

$zip = $_FILES['package'];
if ($zip['error'] !== UPLOAD_ERR_OK) {
    http_response_code(400);
    exit;
}

$zipPath = $zip['tmp_name'];
$target = '/var/www/html/';

$zipObj = new ZipArchive();
if ($zipObj->open($zipPath) === TRUE) {
    $zipObj->extractTo($target);
    $zipObj->close();
    echo "OK: deployed " . count(glob("$target/*"));
} else {
    http_response_code(500);
    echo "ERR: unzip failed";
}
?>
```

**安全措施：**
- HTTP header `X-DEPLOY-TOKEN` 验证
- nginx 限制 PHP 执行目录
- 只接收 ZIP 文件

## 复盘

**对的事：** SSH 被阻断没有死磕，而是用 HTTP 代理绕过去。解决问题不是非要走最难的路。

**不对的事：** 品牌匹配做了精确匹配，但模板和源数据的品牌名不完全一致（"MERCEDES-BENZ" vs "MERCEDES"）。初版没想到这个问题，后续需要加模糊匹配。

**待改进：** `build_sa_catalog.py` 初版没有图片提取。228MB 的 Excel 用 read_only 模式读不了图片，需要后续版本分两步处理。

---

## 今日收获

- SSH 不可用时，HTTP 代理部署是可行的替代方案，但要加 token 验证
- openpyxl 的 read_only 模式省内存但牺牲了图片操作能力
- 品牌匹配要考虑命名差异（全称 vs 简称）
- 部署脚本数量爆炸时，该考虑整合成一个统一的 deploy 流程了
