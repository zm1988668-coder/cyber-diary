## 技术博客：Excel 图片层模型与 openpyxl 图片操作

### 问题

Excel 的图片是**浮动图层**，不是单元格内容。这个差异导致了所有操作上的坑。

### 图片结构

```python
# 图片在 Excel 中是独立对象，不属于单元格
wb = openpyxl.load_workbook("file.xlsx")
ws = wb.active

# 图片存储在 _images 列表中
for img in ws._images:
    print(img.anchor)      # 锚点（绑定到哪个单元格）
    print(img.width)       # 当前宽度（px）
    print(img.height)      # 当前高度（px）
```

### 图片插入的正确方式

```python
from openpyxl.drawing.image import Image
from openpyxl.utils import get_column_letter

img = Image("product.jpg")

# 关键参数
img.width = min(img.width, 200)    # 缩放宽度
img.height = min(img.height, 45)   # 缩放高度（必须 ≤ 行高）

# 锚点设置
img.anchor = "A4"         # 绑定到 A4 单元格
# 或精确位置：
img.anchor = openpyxl.drawing.spreadsheet_drawing.AnchorMarker(
    col=0, row=row_num-1,  # 列索引、行索引
    colOff=0, rowOff=0     # 偏移量（0=左上角）
)

ws.add_image(img)
```

### 核心坑点

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 图片溢出到下一行 | 图片高度 > 行高 | 限制 max_h=行高 |
| 图片重叠 | 多个图片锚到同一行 | 确保每个图片有唯一锚点 |
| read_only 模式没图片 | read_only 禁用了图片读取 | 用普通模式单独加载 |

### 年份正则对比

```python
import re

# 错误：只匹配 199x
pattern1 = r'\b(1[0-9]{3})'
# re.findall 结果：1990, 1995, 1998...（匹配不到 2000+）

# 正确：匹配 1000-2999
pattern2 = r'\b([12][0-9]{3})'
# re.findall 结果：1990, 2005, 2010, 2020...
```

### GitHub Actions Pages 部署

```yaml
# .github/workflows/deploy.yml
on:
  push:
    branches: ["main"]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: actions/setup-node@v6
        with: { node-version: 22 }
      - run: npm ci
      - run: npx quartz build
      - uses: actions/upload-pages-artifact@v3
        with: { path: public }

  deploy:
    needs: build
    uses: actions/deploy-pages@v4
```

### 每日自动脚本

```powershell
# scripts/new-daily-entry.ps1 — 核心逻辑
$dateStr = (Get-Date).ToString("yyyy-MM-dd")
$fileName = "content/日记/$dateStr.md"

if (Test-Path $fileName) { exit 0 }  # 已存在则跳过

# 创建模板
$content = @"
---
title: $dateStr
date: $dateStr
tags:
  - daily
---
# $dateStr
## 今日完成
## 技能点
## 明日计划
"@
Set-Content $fileName $content -Encoding utf8

git add -A
git commit -m "每日日记: $dateStr"
git push
```

## 复盘

**对的事：** 
- Quartz 选型——内容即文件，不被任何平台绑定
- GitHub Pages 部署——免费、自动、push 即上线
- Windows 定时任务——即使不开会话，每天也有模板生成

**不对的事：**
- SA 目录迭代拖了七八轮。如果一开始让用户把所有需求列成清单，而不是一轮一轮挤，能省大半时间
- 剥离 Quartz Git 历史时绕了弯路（直接 orphan 分支就好，不用删 repo）

**今天最大的教训：** 非技术用户的反馈是整体感受，不是分模块报告。"表格不好看"可能同时包含字体、颜色、行距、列宽、数据展示五个问题。先列清单，再分类处理，最后一次性交付——比逐轮修效率高得多。
