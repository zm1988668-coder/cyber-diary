### 全品类车型表项目启动

**用户需求：** 做一个全品类的车型 Excel 表格，包含全球所有品牌和车型。

**需求澄清过程（多轮反问）：**
1. 用户给了一个初步模板（5 列：车型名称、品牌、车型名称、自-至时间、车型描述）
2. 我反问了数据来源、覆盖范围、年份范围、是否需要补充字段
3. 用户明确：
   - 需要全世界的车型，品牌要包含全部车型及年份
   - 同一车型不同年份需要分行
   - 只需要文字描述，不需要补充（级别、动力类型、排量、马力、价格等）
   - 需要停产老车
   - 车型类型包括：皮卡、微面/面包车、轻客、商用货车/客车
   - 年份从 1992 年开始
   - 前 50 个品牌

**分工方式：**
- 创建了 6 个任务分批生成（每批 5-10 个品牌）
- 先写 Python 脚本生成 Excel 结构，再由 AI 逐品牌生成数据
- 5 列结构：车型名称 | 品牌 | 车型名称 | 自-至（时间） | 车型（描述）

## 技术博客：AI 生成大规模结构化数据的方案

### 问题

需要生成 50 个品牌 × 每个品牌几十种车型 × 不同年份 = 数千行结构化数据。纯手工不现实，纯 API 不可靠。

### 方案：分片生成 + Excel 写入

```python
import openpyxl
from openpyxl import Workbook

wb = Workbook()
ws = wb.active
ws.title = "全品类车型表"

# 表头
headers = ["车型名称", "品牌", "车型名称", "自-至（时间）", "车型（描述）"]
ws.append(headers)

# 每品牌生成后立即保存，避免内存溢出
def save_brand(brand_name, rows):
    for row in rows:
        ws.append(row)
    wb.save(f"车型表-{brand_name}.xlsx")
```

### Excel 写入模板

```python
# 车型表基础脚本结构
import openpyxl
from openpyxl.styles import Font, Alignment, PatternFill, Border, Side

def create_car_model_workbook():
    wb = Workbook()
    ws = wb.active
    ws.title = "车型表"
    
    # 表头样式
    header_font = Font(name="Microsoft YaHei", bold=True, size=11)
    header_fill = PatternFill(start_color="4472C4", end_color="4472C4", fill_type="solid")
    header_font_white = Font(name="Microsoft YaHei", bold=True, size=11, color="FFFFFF")
    
    headers = ["车型名称", "品牌", "车型名称", "自-至（时间）", "车型（描述）"]
    for col, h in enumerate(headers, 1):
        cell = ws.cell(row=1, column=col, value=h)
        cell.font = header_font_white
        cell.fill = header_fill
    
    return wb

# 安全写入，防止 Excel 锁定
import os
def safe_save(wb, path):
    if os.path.exists(path):
        base, ext = os.path.splitext(path)
        path = f"{base}_new{ext}"
    wb.save(path)
    return path
```

## 复盘

**对的事：**
- 分品牌分批次生成数据，每批保存，防止崩溃丢数据
- 先反问用户需求再动手，避免方向性错误

**不对的事：**
- 俄罗斯客户精准度不够 — 用户要的是方向机、助力泵、刹车片，但找到的客户不精准。应该先明确关键词再启动搜索
- 德国客户表格格式与俄罗斯不一致 — 应该一开始就确认统一模板

**待改进：** 客户收集流程应该先定义好：搜索关键词 → 渠道 → 数据格式 → 验证方式，再批量执行。而不是先跑起来再发现问题。
