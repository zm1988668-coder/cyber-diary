## 技术博客：为什么汽配独立站选了自建而不是 WordPress

### 问题背景

做汽配 B2B 出口，需要一个网站展示产品和公司实力。市面方案很多。

### 方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| Shopify | 开箱即用、模板多 | 月费、汽配品类限制、B2B不友好 |
| WordPress + WooCommerce | 灵活、生态大 | 插件多就慢、安全维护、慢 |
| 自建前端+API | 轻量、完全控制 | 从头搭需要时间 |

### 决策逻辑

选了自建，不选 WordPress，原因是一句话：**汽配 B2B 不需要购物车。**

客户的典型流程是"看到产品 → 发邮件/WhatsApp 询价 → 谈价格 → 线下成交"，不是"加购物车 → 结账"。所以独立站本质上是一个"产品目录 + 公司展示 + 询盘入口"，不是电商网站。

不需要电商功能，就不要去扛电商框架的复杂度。

### 技术栈

```
前端：HTML + CSS + JS（静态，尽量轻）
后端：Node.js + Express
数据库：PostgreSQL
部署：Nginx 反向代理 → Node 服务
服务器：香港 VPS
```

## 复盘

**对的事：** 选型时没被"流行方案"带偏，从业务实际流程出发做决定。独立站不需要电商功能就扛电商框架，这个判断是对的。

**不对的事：** 第一天花了很多时间找 SVG Logo。这种事其实可以用脚本批量处理，手工一个个下载太慢。

**待验证：** 自建方案在长期维护上比 WordPress 省力还是更费力？目前没有答案。WordPress 有社区支持但需要不断打补丁，自建只要不崩就不用管。

## 今日收获

- 独立站选型由业务模式决定，不是由技术趋势决定
- 微信生态的自动化工具随时可能失效，要有备选方案
- 素材准备工作可以自动化，不要手动下载
