## 技术博客：专有名词翻译为什么不能靠 API

### 问题

1500+ 车型名需要中英对照，第一反应是用翻译 API 批量处理。

### 尝试

```python
# 用翻译 API 的尝试——失败
import translators as ts
result = ts.translate_text("E-Class W211 FL", to_lang="zh")
# 输出: "E-Class W211 FL"（没翻出来）
# 或者翻成: "E级W211 FL"——不完全对
```

### 为什么不行

车型名是"专有名词 + 代号 + 修饰语"的组合结构：
- `Corolla` → 专有名词，翻译 API 能处理
- `E-Class W211 FL` → 专有名词 + 底盘代号 + 改款标记，API 处理不了
- `X5 E70 LCI` → 同理

### 最终方案

半自动：
1. 用 Wikipedia API 批量拉取已知的中文车型名作为基础
2. 剩下的手工查补

```python
# Wikipedia API 提取中文名示例
import requests
url = "https://en.wikipedia.org/w/api.php"
params = {
    "action": "query",
    "titles": "BMW_3_Series_(E90)",
    "prop": "langlinks",
    "lllang": "zh",
    "format": "json"
}
resp = requests.get(url, params=params)
```

### 跨市场差异

同一个品牌在不同市场的中文名不同：Mitsubishi → 三菱（大陆）/ 三菱自動車（台湾）。做数据的时候必须明确"为哪个市场服务"。

## 知识库

### Nginx 反向代理配置参考

```nginx
# Node.js 应用反向代理
server {
    listen 80;
    server_name kitlamt.com;

    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location / {
        root /var/www/html;
        index index.html;
    }
}
```

### 查看 Nginx 配置语法

```bash
# 验证配置是否合法
nginx -t

# 重新加载配置（不中断连接）
nginx -s reload
```

### Wikipedia API 批量查询

```bash
# 获取页面中文翻译
https://en.wikipedia.org/w/api.php?action=query&titles=PAGE_TITLE&prop=langlinks&lllang=zh&format=json
```

## 复盘

**对的事：** 没有强行用自动化解决所有问题。尝试了 API 方案，发现不行就转了半自动 + 人工。

**不对的事：** 车型表的中英对照和交叉比对可以同时做，而不是分两轮。做完一轮才发现漏了 149 行，如果能整合成一次过的工作流，能省一半时间。

**一句话经验：** 数据工作的完成标准不是"所有数据都对了"，而是"关键数据都对了，剩余的不影响业务决策"。

## 今日收获

- 专有名词翻译靠不住 API，半自动 + 人工核验是实际可行的路径
